## TECH-Критерії прийняття проєкту з Python Programming (Neoversity)

## Загальні критерії прийому роботи проєкту


1. Проєкт розташований у загальнодоступному репозиторії на GitHub (можна використати альтернативу таку як GitLab або BitBucket).
2. Наявність коментарів та документації до коду. Присутня докладна інструкція щодо встановлення та використання застосунку описана в файлі Readme.md.
3. Коректність реалізації всіх вимог. Всі вимоги, описані вище, хоча б частково реалізовано.
4. Інтерфейс користувача реалізовано в вигляді командного рядка.
5. Інтерфейс користувача базується на текстових повідомленнях та командах, які користувач вводить з клавіатури.
6. Зручність та логічність інтерфейсу командного рядка.
- Варто вiдображати головне меню лише при запуску програми, а не пiсля кожної запущеної команди, щоб не загромаджувати консоль.
- Інтерфейс повинен бути читабельним та інтуїтивно зрозумілим.
7. Програма взаємодіє з користувачем в циклі, пропонуючи вибрати команду та обробляючи її, поки користувач не введе команду для виходу.
8. Дані коректно зберігаються на жорсткому диску і не втрачаються після перезапуску помічника.
9. Відсутність помилок у коді при виконанні застосунку.
10. Програма повинна коректно обробляти некоректне введення даних користувача без закриття програми.
11. Ефективність використання ООП, спадкування та композиції.
12. Правильна реалізація валідації для кожного поля.
13. Код повинен бути чистим, структурованим та дотримуватися стандартів PEP 8.
14. Ментор провів код-ревью фінальної версії проєкту.

## Критерії прийому додаткової частини проєкту

1. Реалiзовано додавання "тегів" до нотаток.  
2. Реалiзовано пошук та сортування нотаток за "тегами".
